The Extension Layer: 4 Ways to Customize

Click the tabs below to understand the 4 pillars of the Seal Labs AI architecture.

The Constitution ("Always On")

A markdown file at the project root that loads automatically every session. This is where we enforce our "Boring Solutions" rules.

# CLAUDE.md Example


- Use pnpm, not npm

- Follow our 'Boring Solutions' API conventions

- Never hardcode secrets

For Non-Tech: Think of this as your "Brand Voice" document. Always upload it first so the AI knows who we are.

The Toolbox ("On Demand")

Reusable workflows triggered by a specific command. Use this for complex, repetitive tasks like deployment or reviews.

Example Use Cases:

  • /deploy - Triggers the safe deployment checklist.
  • /onboard - Pulls documentation for a new team member.
  • /review - Checks code against style guide.

For Non-Tech: These are your "Saved Prompts." E.g., a prompt that automatically formats a raw meeting transcript into a structured Notion summary.

The Contractors ("Isolated Work")

Isolated workers spawned to perform heavy tasks without bloating your main chat context. Great for research.

Main Context Stays clean and fast.
Subagent Reads 50 files → Returns 1 summary.

For Non-Tech: Use this logic for "Deep Research." Ask the AI to research 10 competitors and only give you the summary table, ignoring the noise.

The Safety Rails ("Automation")

Deterministic scripts that run automatically on events. No AI hallucination involved here.

Key Triggers:

  • PostToolUse: Auto-format code after every AI edit.
  • PreToolUse: Block dangerous commands like rm -rf.
  • Stop: Auto-commit changes when task completes.

For Non-Tech: This is like Zapier/Make automations. E.g., "When a task is marked Done in Lark, automatically email the client."
