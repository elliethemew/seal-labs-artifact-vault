Build Recap

Seal Labs Artifact Vault

An internal hub to browse, search, and reuse HTML artifacts. Built using an AI-assisted outcome-loop workflow.

The Why: Idea → Product

The Problem

We have many internal HTML artifacts, mini-guides, and demos, but they are scattered across various links. They are hard to find and even harder to reuse.

The Solution

Artifact Vault serves as a lightweight internal hub. The MVP scope was locked early: Browse → Find → Open. No heavy admin, no complex features.

  • Browse artifacts like a gallery
  • Search + Filter fast
  • Open detail view + copy link

How I built it: The "Outcome Loop"

The core method was working in small, high-signal loops rather than big specs.

1

Define the next outcome (One slice only)

Example: "Render artifact cards" or "Add filter by tags". Keep it singular.

2

Ask AI for a plan + edge cases

"What components are needed? What states do we need (loading, empty, error)?"

3

Generate code fast with Antigravity

Produce the actual React/Vite implementation quickly.

4

Run locally + Verify behavior

Does search work? Does filter combine? Does the modal close?

5

Screenshot → Critique → Iterate

Send screenshots back to AI to spot UI bugs or UX friction I missed.

Only commit when the slice is stable end-to-end.

Tool Roles

GPT

  • • Provided structured checklists & sanity checks.

What I Learned: The AI Working Rules

Clarity is King

AI works best when I give one clear outcome, not a big vague request.

Verify, Don't Trust

I get better results when I verify + screenshot + iterate, instead of trusting the first output.

Speed via Slices

Speed comes from splitting work into "small slices" and shipping each slice fully.
